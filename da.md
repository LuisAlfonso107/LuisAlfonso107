### Historia de Usuario US-026
**Crear un README.md super cool, oscuro y futurista con estilo hacker demoníaco**

**Como** desarrollador frontend  
**Quiero** generar un README.md impactante, oscuro y con personalidad fuerte para mi portafolio  
**Para** que al abrir mi repositorio la gente sienta inmediatamente que está entrando en algo diferente, oscuro y de alto nivel técnico

**Criterios de Aceptación**

**Escenario 1: Estilo general del README**
Dado que alguien abre el repositorio  
Cuando ve el README.md  
Entonces debe transmitir:

- Atmósfera oscura, hacker y ligeramente demoníaca (sin caer en clichés baratos tipo "bienvenido al futuro")
- Estilo visual premium y cinematográfico
- Combinación equilibrada entre texto, imágenes, badges, animaciones y bloques de código
- Sensación de que lo escribió una persona real con personalidad fuerte, no una IA genérica

**Escenario 2: Animaciones y elementos visuales geniales**
El README debe incluir animaciones y efectos visuales de alto impacto:
- Typing effect oscuro y elegante (no frases ridículas)
- Imágenes grandes con estilo oscuro y perturbador
- Badges y shields con tema oscuro consistente
- Tablas y mermaid diagrams con tema dark personalizado
- Contadores o stats con animación sutil
- Contribución graph personalizado (pacman, snake, o similar con tema oscuro)
- ASCII art o arte vectorial oscuro si encaja con el estilo

**Escenario 3: Estructura y contenido**
El README debe contener las siguientes secciones de forma natural y bien organizada:

- Header impactante con nombre y título
- Imagen principal o banner oscuro
- Manifiesto o introducción personal (corta y potente)
- Perfil profesional con foto y datos clave
- Especializaciones / Stack técnico (con badges bien diseñados)
- Arquitectura visual (Mermaid diagram oscuro)
- Estadísticas de GitHub (con tema oscuro)
- Contribuciones (gráfico personalizado)
- Fragmentos de código destacados (con sintaxis highlighting oscuro)
- Actividad reciente
- Contacto con badges elegantes
- Sección final fuerte (últimas palabras o pacto)

**Escenario 4: Calidad y coherencia**
- Todo el README debe verse profesional pero con alma oscura y carácter
- Evitar frases cliché como "bienvenido al futuro", "demon engineer", "abismo te observa", etc.
- El tono debe ser misterioso, técnico y con personalidad propia
- Excelente legibilidad a pesar del estilo oscuro
- Totalmente responsive (se ve bien en móvil y desktop)

**Notas**
- El estilo debe ser oscuro, hacker y futurista pero elegante
- Usar animaciones modernas (typing svg, github stats, contribution graphs, etc.)
- El README debe sentirse escrito por una persona con identidad fuerte, no por una IA genérica
- Puedes usar cualquier recurso (shields.io, readme-typing-svg, github-readme-stats, mermaid, etc.)
- El objetivo es que sea visualmente impresionante y memorable
